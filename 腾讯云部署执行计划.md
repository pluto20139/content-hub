# 腾讯云部署执行计划

> **目标**：将 Cron 从 GitHub Actions 迁移到腾讯云服务器（国内），前端 H5 + Admin 从 Vercel 迁到腾讯云 nginx，删除 bilibili-fetch Edge Function（国内直连 B站无需代理）。
>
> **决策摘要**：
> - 服务器：腾讯云国内（需备案）
> - Cron 部署：pm2 进程管理
> - bilibili-fetch：删除（国内直连 B站 API）
> - 前端：腾讯云 nginx 托管 H5 + Admin
>
> **预计总耗时**：4-6 小时（不含备案等待）

---

## 架构变更对比

### Before（当前）

```
GitHub Actions (每 30 min) → Cron 脚本 → Supabase Cloud (新加坡)
                            ↓
                            bilibili-fetch Edge Function (代理) → B站 API
                            ↓
                            YouTube API

Vercel → H5 SPA
Vercel → Admin SPA
```

### After（目标）

```
腾讯云服务器 (国内, pm2) → Cron 脚本 (每 30 min) → Supabase Cloud (新加坡)
                            ↓ 直连
                            B站 API ✅ (国内直连无需代理)
                            ↓
                            YouTube API ✅ (国内可访问 googleapis)

腾讯云 nginx → H5 SPA (HTTPS + 备案域名)
腾讯云 nginx → Admin SPA
```

### 关键变化

| 项 | Before | After | 影响 |
|---|---|---|---|
| Cron 运行环境 | GitHub Actions ubuntu | 腾讯云国内服务器 | B站直连无需代理 |
| bilibili-fetch | Edge Function 代理 | 删除 | SSRF 风险消除，架构简化 |
| BILIBILI_PROXY_URL | 必配 | 删除 | secrets 减一项 |
| CRON_API_KEY | 必配 | 删除 | secrets 减一项 |
| 前端 H5 | Vercel | 腾讯云 nginx | 需备案域名 + SSL |
| 前端 Admin | Vercel | 腾讯云 nginx | 同上 |
| Cron 调度 | GitHub schedule cron | pm2 + node-cron 或 system crontab | 调度逻辑本地化 |
| RUN_ID | `run-${GITHUB_RUN_ID}` | `run-${Date.now()}` | 已有 fallback，无需改 |

### 1.4 Cron 调度改为 pm2 + node-cron

**新建文件**：`scripts/cron/src/scheduler.ts`

```typescript
import cron from "node-cron";
import { run } from "./index.js";

const CRON_SCHEDULE = "*/30 * * * *"; // 每 30 分钟

async function main(): Promise<void> {
  console.log(`[SCHEDULER] Cron scheduled: ${CRON_SCHEDULE}`);

  // 启动时立即跑一次
  await run().catch((err) => console.error("[SCHEDULER] Initial run failed:", err));

  // 注册定时任务
  cron.schedule(CRON_SCHEDULE, async () => {
    console.log(`[SCHEDULER] Tick at ${new Date().toISOString()}`);
    try {
      await run();
    } catch (err) {
      console.error("[SCHEDULER] Run failed:", err);
    }
  });
}

main().catch((err) => {
  console.error("[SCHEDULER] Fatal:", err);
  process.exit(1);
});
```

### 4.3 配置 nginx

**服务器上创建**：`/etc/nginx/sites-available/content-hub`

```nginx
# H5 站点（主域名）
server {
    listen 80;
    server_name contenthub.example.com;  # 改成你的域名
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name contenthub.example.com;  # 改成你的域名

    ssl_certificate     /etc/nginx/ssl/contenthub.crt;  # 改成你的证书路径
    ssl_certificate_key /etc/nginx/ssl/contenthub.key;

    root /opt/content-hub/apps/h5/dist;
    index index.html;

    # SPA fallback：所有路由都返回 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|svg|png|jpg|jpeg|gif|ico|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
}

# Admin 站点（子域名或子路径）
server {
    listen 443 ssl http2;
    server_name admin.contenthub.example.com;  # 改成你的子域名

    ssl_certificate     /etc/nginx/ssl/contenthub.crt;
    ssl_certificate_key /etc/nginx/ssl/contenthub.key;

    root /opt/content-hub/apps/admin/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|svg|png|jpg|jpeg|gif|ico|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Admin 不应被搜索引擎收录
    add_header X-Robots-Tag "noindex, nofollow";

    # IP 白名单（可选，加强 Admin 安全）
    # allow YOUR_OFFICE_IP;
    # deny all;
}
```

## 风险与回滚

### 风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 备案被拒 | 低 | 阻断前端上线 | 用香港服务器备选，或前端留 Vercel |
| 服务器 IP 被 B站 封 | 低 | Cron 抓取失败 | 保留 bilibili-fetch 代码，需要时重新部署 |
| pm2 进程崩溃 | 中 | Cron 停跑 | pm2 autorestart + 开机自启 |
| Supabase 跨境延迟 | 低 | Cron 慢 200-500ms | 可接受，Cron 不是实时 |
| 国内访问 googleapis | 中 | YouTube 抓取失败 | 服务器装代理或 YouTube API 走 Edge Function |

### YouTube API 国内访问问题

**注意**：腾讯云国内服务器访问 `www.googleapis.com`（YouTube Data API）可能不稳定。**这是本次迁移的最大风险**。

**预检**：Batch 2 完成后立即在服务器上测：
```bash
curl -I https://www.googleapis.com/youtube/v3/
```
- 200/401/403 → 可访问（401/403 是缺 API key，但说明能连通）
- 超时/连接重置 → 需要代理

**如果 YouTube API 不可达**：
- 方案 A：服务器装 HTTP 代理（如 v2ray），YouTube 抓取走代理，B站 直连
- 方案 B：YouTube 抓取保留在 Supabase Edge Function（新加坡机房可访问 googleapis）
- 方案 C：YouTube 改用 `youtubei.js`（无需 API key，但违反 YouTube ToS）

### 回滚方案

如果腾讯云部署失败：
1. Cron 回滚：恢复 `.github/workflows/cron-fetch.yml`，重新启用 GitHub Actions
2. 前端回滚：Vercel 部署保留，DNS 切回 Vercel
3. bilibili-fetch 回滚：恢复 Edge Function 代码 + 重新部署

**git 层面**：所有改动在单独 commit / branch，回滚 = `git revert`。

## V1.1 迭代版本部署说明

### 1. 自托管 RSSHub 部署 (Docker Compose)
知乎、抖音和小红书的内容抓取依赖于 RSSHub 服务。由于公共实例风控极高，建议在云服务器上通过 Docker Compose 进行自托管：
```yaml
version: '3'

services:
  rsshub:
    image: diygod/rsshub:latest
    restart: always
    ports:
      - '1200:1200'
    environment:
      NODE_ENV: production
      CACHE_TYPE: redis
      REDIS_URL: redis://redis:6379/0
      # 鉴权访问限制（SPEC AC5）
      ACCESS_KEY: your_rsshub_access_key
      # 小红书抓取所需的全局 Cookie
      XIAOHONGSHU_COOKIE: 'your_cookie_here'
    depends_on:
      - redis

  redis:
    image: redis:alpine
    restart: always
    volumes:
      - redis-data:/data

volumes:
  redis-data:
```

### 3. 风控应对机制 (平台级短路)
当抖音、小红书等平台在抓取时识别到防爬墙、验证码（Captcha）或者 403 触发时，适配器会捕获并自动上报 `PlatformLevelError`，将数据库中的平台状态更新为 `paused:until=<now+30min>` 并暂停该平台 30 分钟，以有效保护爬虫 IP。
