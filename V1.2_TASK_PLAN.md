### T1：数据库迁移

**文件**：新建 `supabase/migrations/015_v1_2_summary_fields.sql`

```sql
ALTER TABLE contents ADD COLUMN summary            TEXT,
                      ADD COLUMN summary_status    VARCHAR(20) DEFAULT NULL,
                      ADD COLUMN summary_at        TIMESTAMPTZ DEFAULT NULL,
                      ADD COLUMN summary_duration_ms INT       DEFAULT NULL;

CREATE INDEX idx_contents_summary_pending
  ON contents (summary_status)
  WHERE summary_status = 'pending';
```

### T4：Cron 新增 Dify 总结模块

**文件**：新建 `scripts/cron/src/lib/dify.ts`

**功能**：
```ts
interface SummaryTask {
  contentId: number;
  title: string;
  url: string;
  platform: string;
  contentType: string;
}

// 并发池（≤5，不遗漏）
async function processSummaryQueue(items: SummaryTask[], concurrency?: number): Promise<void>

// 单条 Dify 调用（30s 超时 + 1 次重试 + 5s 间隔）
async function triggerDifySummary(task: SummaryTask): Promise<void>
```

**Dify 调用逻辑**：
1. UPDATE contents SET summary_status='pending' WHERE id=contentId
2. POST Dify Workflow API（30s 超时）
3. 成功 → UPDATE summary, summary_status='completed', summary_at=now(), summary_duration_ms=耗时
4. 失败 → sleep 5s → 重试 1 次
5. 重试失败 → UPDATE summary_status='failed'

**验收**：单元测试 mock Dify API，验证并发池不遗漏、重试逻辑正确。

### T5：Cron index.ts 集成 Dify 总结队列
